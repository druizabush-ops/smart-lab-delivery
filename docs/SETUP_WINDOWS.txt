7. Текущий запуск приложения
На текущем этапе полноценный runtime отсутствует: проект находится в фазе инженерной и архитектурной инициализации.

8. Что проверять на этом этапе
- Наличие обязательных документов в docs/.
- Соответствие структуры каталогов архитектурным слоям.
- Наличие записи в PROJECT_JOURNAL о произведенных изменениях.
- Отсутствие преждевременной реализации интеграционных клиентов, БД, UI и доставки.

9. Установка и запуск тестов (pytest)
```powershell
python -m pip install --upgrade pip
python -m pip install pytest
```

10. Запуск тестов в PowerShell
```powershell
# Все тесты
python -m pytest

# Только unit-тесты domain
python -m pytest .\tests\unit\domain

# Только unit-тесты application
python -m pytest .\tests\unit\application

# Только integration-тесты
python -m pytest .\tests\integration
```

11. Как читать результат
- `passed` — сценарий отработал по ожидаемому контракту.
- `failed` — зафиксирован регресс или нарушение инварианта.
- `xfailed/skipped` сейчас не используются в базовом контуре.
- Для детального вывода используйте:
```powershell
python -m pytest -vv
```

12. Запуск runtime cycle (PowerShell)
```powershell
# Один синхронный цикл runtime (seed + processing)

```
Ожидаемый результат: вывод блока `Delivery Runtime Summary` с числом созданных/обработанных карточек и финальными счетчиками.

14. Важные ограничения текущего runtime
- Хранилище и очередь in-memory: состояние не сохраняется после завершения процесса.
- Execution loop синхронный: фоновый scheduler/worker не реализован.
- Integration-клиенты остаются stub-адаптерами.

21. Запуск operator API (read-only) в PowerShell
```powershell

# Вариант 1: прямой запуск скрипта из корня проекта

```
Ожидаемо: оба способа поднимают Uvicorn на `http://127.0.0.1:8001`.

26. Запуск process manager (scheduler lifecycle) в PowerShell
```powershell
# Полный pending lifecycle: новые READY + retry/pending карточки
python .\scripts\run_process_manager.py
```
Ожидаемо: вывод блока `Delivery Process Manager Summary` с полями:
`new_cards_discovered`, `existing_cards_reprocessed`, `retry_candidates_count`, `processed_count` и финальными счетчиками исходов.

27. Локальная проверка retry lifecycle
```powershell
# 1) Первый цикл создает карточки и фиксирует начальные статусы
python .\scripts\run_process_manager.py

# 2) Повторный цикл переобрабатывает pending/retry-кандидатов из persistence
python .\scripts\run_process_manager.py

# 3) Прогон scheduler/process manager тестов
python -m pytest .\tests\unit\infrastructure\test_delivery_process_manager.py
```

28. Ограничения process manager текущего этапа
- Режим синхронный, без background thread/async worker.
- Нет cron и нет broker-based orchestration.
- Process manager не принимает бизнес-решения retry/fallback/deduplication; решения остаются в application policy/use-case слое.

29. Переключение real integration mode (PowerShell)
```powershell
# Включить реальные интеграции
$env:SLD_INTEGRATION_MODE = "real"

# Renovatio
$env:SLD_RENOVATIO_BASE_URL = "https://<renovatio-endpoint>"
$env:SLD_RENOVATIO_API_KEY = "<secret>"
$env:SLD_RENOVATIO_API_VERSION = "1"
$env:SLD_RENOVATIO_PATIENT_IDS = "patient-001,patient-002"

# MAX
$env:SLD_MAX_BASE_URL = "https://platform-api.max.ru"
$env:SLD_MAX_BOT_TOKEN = "<secret>"
$env:SLD_MAX_BOT_NAME = "<botName>"
$env:SLD_MAX_PATIENT_RECIPIENT_MAP = "patient-001:max-user-1,patient-002:max-user-2"
```

31. Откат на stub-режим
```powershell
$env:SLD_INTEGRATION_MODE = "stub"
Remove-Item Env:SLD_RENOVATIO_API_KEY -ErrorAction SilentlyContinue
Remove-Item Env:SLD_MAX_BOT_TOKEN -ErrorAction SilentlyContinue
Remove-Item Env:SLD_EMAIL_PASSWORD -ErrorAction SilentlyContinue
```

32. Ограничения текущего блока real integrations
- Реальные live-вызовы в тестах CI не выполняются.
- MAX identity mapping пока конфигурируется вручную через env-карту.
- Mini app UI не реализован, зафиксирован только deep-link контракт.

30. Проверка operator command API локально
```powershell
# Перезапустить API
python .\scripts\run_operator_api.py

# Retry карточки
Invoke-RestMethod -Uri "http://127.0.0.1:8001/operator/cards/<card_id>/retry" -Method POST

# Перевод в manual_review
Invoke-RestMethod -Uri "http://127.0.0.1:8001/operator/cards/<card_id>/manual-review" -Method POST -ContentType "application/json" -Body '{"reason":"ручная проверка"}'

# Возврат в очередь
Invoke-RestMethod -Uri "http://127.0.0.1:8001/operator/cards/<card_id>/requeue" -Method POST -ContentType "application/json" -Body '{"reason":"повторная обработка"}'

# Override канала
Invoke-RestMethod -Uri "http://127.0.0.1:8001/operator/cards/<card_id>/override-channel" -Method POST -ContentType "application/json" -Body '{"channel":"EMAIL","reason":"MAX недоступен"}'
```
