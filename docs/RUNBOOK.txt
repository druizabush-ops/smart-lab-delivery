Название: Runbook эксплуатации Smart Lab Delivery
Дата актуализации: 2026-04-14

1. Система не стартует
Шаги:
1) Проверить env-контракт: `python scripts/run_launch_readiness_checks.py`.
2) Проверить traceback старта API/скрипта.
3) Проверить обязательные env: `SLD_DATABASE_URL`, `SLD_MAX_BOT_TOKEN`, `SLD_RENOVATIO_API_KEY`.
4) Для быстрой стабилизации в non-prod допускается переключение в safe-mode:
   - `SLD_INTEGRATION_MODE=stub`
   - `SLD_REPOSITORY_MODE=in_memory` (только если допустима потеря state)

2. БД недоступна
Симптом: `/health/ready` -> `checks.db=false`.
Шаги:
1) Проверить доступность PostgreSQL хоста/порта.
2) Проверить `SLD_DATABASE_URL` и креды.
3) Выполнить `python -m alembic current`.
4) После восстановления БД повторить `python scripts/run_launch_readiness_checks.py`.

3. health/ready red
Симптом: `status=not_ready`.
Шаги:
1) Проверить `checks` в payload (`db`, `config`, `integration_mode`).
2) Если `config=false` — проверить `SLD_ENV`, `SLD_REPOSITORY_MODE`, `SLD_INTEGRATION_MODE`.
3) Если `integration_mode=false` — восстановить допустимое значение `stub|real`.
4) Если `db=false` — следовать сценарию из раздела 2.

4. MAX transport упал
Шаги:
1) Проверить `SLD_MAX_BOT_TOKEN`, `SLD_MAX_BOT_NAME`, `SLD_MAX_BASE_URL`.
2) Проверить `SLD_MAX_PATIENT_RECIPIENT_MAP`.
3) Временная деградация: оставить email fallback активным.
4) Если канал нестабилен длительно — временно `SLD_INTEGRATION_MODE=stub` (только по решению ответственного релиза).

5. Email transport недоступен
Шаги:
1) Проверить SMTP host/port/TLS.
2) Проверить логины/пароли и from-адрес.
3) Подтвердить, что MAX канал работает как primary.
4) При длительной аварии зафиксировать ограничение в KNOWN_LIMITATIONS и журнале смены.

6. Patient API отдаёт ошибки
Шаги:
1) Проверить `SLD_PATIENT_SECURITY_MODE` и корректность initData context.
2) Проверить `/health/ready`.
3) Проверить, что patient_id не подменяется клиентом без проверки подписи.
4) Временно ограничить выдачу через strict policy + диагностика логов correlation-id.

7. Operator command не проходит
Шаги:
1) Проверить, что состояние карточки допускает команду по OperatorActionPolicy.
2) Проверить код ответа: 409 ожидаем при policy reject.
3) Проверить наличие записи в audit trail.
4) При массовом reject — проверить корректность операционных инструкций.

8. Audit trail не пишет
Шаги:
1) Проверить режим repository (`postgres` обязателен для persistence audit).
2) Проверить наличие таблицы `operator_action_log`.
3) Проверить, что operator command use case получает action_logger.
4) До восстановления запретить ручные командные действия вне runbook.

9. Миграция не прошла
Шаги:
1) Остановить rollout новых процессов.
2) Зафиксировать ошибку миграции и ревизию.
3) Откатить последнюю миграцию: `python -m alembic downgrade -1`.
4) Восстановить предыдущую стабильную ревизию и повторить smoke.

10. Возврат на safe mode (минимальный rollback)
Допустимо только по решению ответственного релиза и вне prod:
- `SLD_INTEGRATION_MODE=stub`
- `SLD_PATIENT_SECURITY_MODE=strict` сохраняется
- Перезапуск API + `python scripts/run_launch_readiness_checks.py`

11. Единый admin flow через ZAPUSK228
Шаги:
1) Базовый запуск с подтверждениями:
   - `python scripts/zapusk228.py`
2) Неинтерактивный dry-run:
   - `python scripts/zapusk228.py --assume-yes --dry-run`
3) Пример частичного запуска без frontend/restart:
   - `python scripts/zapusk228.py --assume-yes --skip-frontend --skip-restart`

Примечания:
- ZAPUSK228 не печатает секреты env, только факт наличия ключевых переменных.
- Для server-операций с systemd запускать инструмент от пользователя с нужными правами.

12. Process manager start-limit-hit
Симптом: сервис `smart-process-manager` уходит в `start-limit-hit` после успешного выполнения цикла.
Проверка:
1) `systemctl cat smart-process-manager.service`
2) Убедиться, что используется one-shot модель (`Type=oneshot`, `Restart=no`).
3) Убедиться, что периодичность обеспечивается `smart-process-manager.timer`, а не `Restart=always`.

13. Patient PDF endpoint возвращает 404/502
Симптомы:
- `GET /patient/results/{result_id}/pdf` -> 404 (нет файла) или 502 (невалидный payload).
Шаги:
1) Проверить активность session через `GET /patient/auth/me` (должен быть 200).
2) Проверить детали результата `GET /patient/results/{result_id}` и наличие признаков документов.
3) Проверить интеграционный payload `getPatientLabResultDetails`: наличие `data.files` и корректность base64 в `files[0]`.
4) При системной проблеме Renovatio временно зафиксировать инцидент и переключиться на ручной канал выдачи PDF по операционному регламенту.

14. ZAPUSK228: опциональный PDF smoke
Для включения smoke проверки PDF endpoint в `scripts/zapusk228.py` перед запуском задайте:
- `SLD_SMOKE_PATIENT_SESSION`
- `SLD_SMOKE_PATIENT_RESULT_ID`

Пример:
- `SLD_SMOKE_PATIENT_SESSION=<session_id> SLD_SMOKE_PATIENT_RESULT_ID=<result_id> python scripts/zapusk228.py --assume-yes --skip-update --skip-deps`

15. Mini app login/results smoke через ZAPUSK228
Для опционального smoke login/results задайте:
- `SLD_SMOKE_PATIENT_LOGIN`
- `SLD_SMOKE_PATIENT_PASSWORD`
И запустите `python scripts/zapusk228.py --assume-yes --skip-update --skip-deps`.

16. Ручная замена mini app assets
Шаги:
1) Подготовить файл с тем же именем и подходящим размером.
2) Положить файл в `frontend/miniapp/public/assets`.
3) Выполнить `cd frontend/miniapp && npm run build`.
4) Задеплоить новый `dist` стандартным frontend flow или через ZAPUSK228.
5) Проверить на телефоне login/home/results/details/pdf и bottom navigation.

Текущие runtime assets:
- `logo-smart.svg` — SVG, ориентир 300x120.
- `avatar-placeholder.svg` — SVG, квадрат 256x256.
- `icon-home.svg`, `icon-analyses.svg`, `icon-appointment.svg`, `icon-promos.svg`, `icon-services.svg` — SVG, viewport 24x24 или 32x32 допустим.
- `icon-eye.svg`, `icon-eye-off.svg` — SVG для password visibility.
- `login-bg.png` или replacement `login-bg.webp` — 1080x1600 или 1170x2532.
- `card-analyses.png`, `card-appointment.png`, `card-promos.png`, `card-services.png` или replacement `.webp` — 600x240 / 800x320.

Важно:
- При замене PNG на WEBP обновить ссылку в `frontend/miniapp/src/App.tsx` или сохранить используемое имя.
- Не использовать screenshot всего Figma screen как фон приложения.
- После замены обязательно проверить, что изображения не перекрывают текст и не ломают mobile layout.
