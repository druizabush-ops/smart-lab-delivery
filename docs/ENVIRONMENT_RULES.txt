Название: Environment rules для Smart Lab Delivery
Дата актуализации: 2026-04-14

1. Общие правила
- Секреты не хранятся в коде, git и документах с реальными значениями.
- Подключение к БД и внешним интеграциям настраивается только через переменные окружения.
- Для локальной разработки допускаются примеры значений-заглушек.

2. Режимы приложения
- SLD_REPOSITORY_MODE: `in_memory` или `postgres`.
- SLD_INTEGRATION_MODE: `stub` или `real`.

3. Обязательные переменные (PostgreSQL)
- SLD_DB_HOST
- SLD_DB_PORT
- SLD_DB_NAME
- SLD_DB_USER
- SLD_DB_PASSWORD
- SLD_DATABASE_URL (приоритетнее host/port/name/user/password)

4. Renovatio env (real mode)
- SLD_RENOVATIO_BASE_URL (обязательно для real).
- SLD_RENOVATIO_API_KEY (обязательно для real).
- SLD_RENOVATIO_API_VERSION (опционально, по умолчанию `1`).
- SLD_RENOVATIO_TIMEOUT_SECONDS (опционально, по умолчанию `10`).
- SLD_RENOVATIO_PATIENT_IDS (опционально, CSV patient_id для чтения ready flow).

5. MAX env (real mode)
- SLD_MAX_BASE_URL (опционально, по умолчанию `https://platform-api.max.ru`).
- SLD_MAX_BOT_TOKEN (обязательно для real).
- SLD_MAX_TIMEOUT_SECONDS (опционально, по умолчанию `5`).
- SLD_MAX_BOT_NAME (опционально, нужен для deep link).
- SLD_MAX_PATIENT_RECIPIENT_MAP (опционально, формат `patient-1:max-user-1,patient-2:max-user-2`).

6. Email env (real mode)
- SLD_EMAIL_SMTP_HOST (обязательно для real).
- SLD_EMAIL_SMTP_PORT (опционально, по умолчанию `25`).
- SLD_EMAIL_USERNAME (опционально).
- SLD_EMAIL_PASSWORD (опционально, обязателен если используется login).
- SLD_EMAIL_FROM (опционально, по умолчанию `no-reply@smart-lab.local`).
- SLD_EMAIL_USE_TLS (опционально: `1`/`0`, по умолчанию `0`).
- SLD_EMAIL_TIMEOUT_SECONDS (опционально, по умолчанию `10`).

7. Рекомендуемые значения для локального dev (PowerShell)
```powershell
$env:SLD_REPOSITORY_MODE = "postgres"
$env:SLD_INTEGRATION_MODE = "stub"
$env:SLD_DB_HOST = "localhost"
$env:SLD_DB_PORT = "5432"
$env:SLD_DB_NAME = "smart_lab_delivery"
$env:SLD_DB_USER = "postgres"
$env:SLD_DB_PASSWORD = "postgres"
$env:SLD_DATABASE_URL = "postgresql+psycopg://postgres:postgres@localhost:5432/smart_lab_delivery"
```

8. Ограничения безопасности
- Реальные ключи Renovatio/MAX/SMTP запрещено коммитить.
- Для локальной проверки real-режима использовать отдельные тестовые учетные данные.

9. Frontend mini app env
- VITE_PATIENT_API_BASE_URL — base URL patient API (по умолчанию `http://127.0.0.1:8001`).
- Для локального запуска вне MAX допускается отсутствие `window.WebApp`; UI должен работать в fallback-режиме.

10. MAX mini app context notes
- `start_param` рассматривается как недоверенный входной payload до серверной валидации.
- `initDataUnsafe.user.id` используется только как временный локальный идентификатор для read-flow.

11. Production hardening env (обновление 2026-04-14)
Обязательные для prod-like:
- SLD_ENV=prod
- SLD_PATIENT_SECURITY_MODE=strict
- SLD_MAX_WEBAPP_SECRET (секрет проверки initData/WebAppData)
- SLD_RATE_LIMIT_ENABLED=1
- SLD_RATE_LIMIT_PER_MINUTE (>0)

Безопасные настройки:
- В prod запрещено оставлять пустыми real credentials (`SLD_RENOVATIO_API_KEY`, `SLD_MAX_BOT_TOKEN` при real mode).
- В prod запрещено использовать `SLD_PATIENT_SECURITY_MODE=relaxed`.

Допустимые dev-only defaults:
- `SLD_PATIENT_SECURITY_MODE=relaxed`.
- Заглушечные `SLD_*_TOKEN/API_KEY` в stub режиме.

12. Final launch env contract (обновление 2026-04-14)
Профили:
- dev/test: допускаются `in_memory` и `stub`, предупреждение о неперсистентном режиме.
- staging/prod (release-like):
  - обязательно `SLD_REPOSITORY_MODE=postgres`;
  - обязательно `SLD_INTEGRATION_MODE=real`;
  - обязательно `SLD_PATIENT_SECURITY_MODE=strict`.

Критичные env для release-like:
- SLD_DATABASE_URL
- SLD_RENOVATIO_API_KEY
- SLD_MAX_BOT_TOKEN
- SLD_MAX_BOT_NAME
- SLD_EMAIL_SMTP_HOST
- SLD_EMAIL_FROM
- SLD_MAX_WEBAPP_SECRET

Unsafe defaults (запрещены в release-like):
- SLD_DATABASE_URL с `postgres:postgres`.
- SLD_EMAIL_SMTP_HOST=localhost.
- Заглушечные example URL для внешних интеграций.
