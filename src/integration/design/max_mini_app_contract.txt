Название: Контракт mini app / deep link интеграции MAX
Дата актуализации: 2026-04-14

1. Цель
Зафиксировать transport-контракт открытия результата из MAX без реализации UI на текущем этапе.

2. Формат deep link
- Базовый формат: `https://max.ru/<botName>?startapp=<payload>`.
- Текущий payload-контракт: `lab_result:<lab_result_id>|patient:<patient_id>`.
- Источник botName: env `SLD_MAX_BOT_NAME`.

3. Что ожидается в mini app
- Открытие web-приложения внутри MAX-клиента.
- На UI-этапе payload читается как входной контекст для загрузки карточки результата.
- Для runtime взаимодействия планируется использовать `window.WebApp` и MAX Bridge.

4. Границы текущего этапа
- UI mini app не реализуется.
- Авторизация mini app пользователя не реализуется.
- Контракт фиксирует только формат входа и передачу контекста.

5. Ограничения
- Без identity mapping между patient_id и MAX user/chat id deep-link не гарантирует доставку конкретному получателю.
- Контракт payload может быть расширен после согласования требований безопасности.
