# smart-lab-delivery

Система автоматизации доставки результатов анализов пациентам клиники.

## Назначение
Проект находится на этапе формирования инженерного каркаса: зафиксирована базовая архитектура, статусная модель, правила документации и сопровождения.

## Структура каталогов
- `docs/` — обязательная инженерная документация и непрерывный журнал проекта.
- `src/integration/renovatio/` — интеграция с МИС Renovatio (клиенты API, адаптеры, нормализация).
- `src/application/` — прикладная оркестрация очереди и доставки.
- `src/domain/` — доменные сущности, контракты и централизованные статусы.
- `src/infrastructure/` — реализации каналов доставки, персистентность, логирование.
- `src/presentation/operator_panel/` — слой операторского интерфейса.
- `tests/` — unit/integration/smoke сценарии.
- `scripts/` — служебные скрипты для запуска и сопровождения.
- `config/` — конфигурационные шаблоны без секретов.

## Текущий фокус
Сначала фиксируем архитектурную дисциплину и правила развития, затем переходим к реализации бизнес-логики.
