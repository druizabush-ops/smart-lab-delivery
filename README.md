# smart-lab-delivery

Система автоматизации доставки результатов лабораторных исследований пациентам клиники.

## Что это за проект
`smart-lab-delivery` — внутренний сервис, который получает данные из МИС Renovatio, формирует карточки доставки и управляет отправкой результатов пациентам.

## Текущая стадия
Проект находится на стадии **инженерного каркаса** (foundation stage):
- слоистая структура каталогов уже сформирована;
- базовая архитектурная документация уже введена;
- стартовая модель статусов уже централизована в документах;
- реализация сложной бизнес-логики, интеграционных клиентов, БД и UI **осознанно отложена** на следующий этап.

## Главный источник инженерной памяти
Основной непрерывный журнал проекта: `docs/PROJECT_JOURNAL.txt`.

Любое существенное изменение должно сопровождаться записью в журнале с причинами, ограничениями и следующими шагами.

## Обязательные документы сопровождения
- `docs/PROJECT_JOURNAL.txt` — непрерывная инженерная память проекта.
- `docs/ARCHITECTURE_OVERVIEW.txt` — целевая архитектурная рамка и инварианты.
- `docs/STATUS_RULES.txt` — единая модель статусов и переходов.
- `docs/DECISIONS.txt` — реестр архитектурных решений в формате ADR.
- `docs/TECH_DEBT.txt` — реестр временных решений и ограничений.
- `docs/SETUP_WINDOWS.txt` — инструкции запуска и проверки под Windows 10 / PowerShell.

## Согласованная структура каталогов
- `src/domain`
- `src/application`
- `src/integration`
- `src/infrastructure`
- `src/presentation`
- `tests`
- `scripts`
- `config`

## Принципы текущего этапа
- Никаких скрытых архитектурных изменений.
- Никаких неявных допущений: все ограничения фиксируются в документах.
- Временные решения помечаются как технический долг.
- Сначала качество инженерной базы, затем переход к функциональным блокам.
